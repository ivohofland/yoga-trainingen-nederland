# Project Decisions & Known Gaps

*Decision log following the helicopter review (2026-06-12). Companion to `data-model-spec.md` and `yoga-opleidingen-directory-overwegingen.md`.*

| # | Topic | Decision | Status |
|---|---|---|---|
| 1 | Reader / distribution strategy | Real gap, consciously deferred. No distribution plan yet; revisit before or around first publication. Candidate wedge: explainer content ("wat betekent 200 uur?", "is CRKBO een keurmerk?") that funnels into the directory. | **Known gap — parked** |
| 2 | Scope / inclusion criteria | Start with official 200/300/500-hour teacher trainings (the YA format categories). Smaller formats (50/85/100hr specializations as standalone offerings) added later; YACEP claim verification later. Enumeration of the provider universe (YA directory, VYN list, CRKBO register, Maps) is an open task. | **Decided — phased** |
| 3 | Legal operations & AVG | Operate like a publication from day one: published complaints/correction procedure, defined right-of-reply window, AVG awareness for the Person entity (journalistic exemption requires actually behaving journalistically), local PDF snapshots alongside Wayback, consider media liability insurance before layer 3 goes live. | **Decided — adopt** |
| 4 | Sustainability / maintenance | Hobby project; no year-2 commitment can be made now. Accepted risk. Mitigation available if needed later: archive entries that can no longer be verified rather than letting `last_verified` decay silently. | **Accepted risk** |
| 5 | Methodology page | Founding deliverable. Must exist publicly before the first critical/assessed content is published. Versioned (`methodology_version` in the schema points at it). | **Decided — sequenced before publication** |
| 6 | Goodhart / metric gaming | Anticipated from the start: once metrics matter, schools will market them and claims will inflate. Defense is structural: verbatim quotes + mandatory sources, claims never stored as fact, and the transparency meta-axis (hardest to game). Schema already encodes this; treat rising verification pressure as a success signal. | **Decided — designed in** |
| 7 | Reader corrections + public change history | In from the beginning. Lightweight "meld een fout" route; validated reader reports enter the dataset as a source (`reader_report` source type). Per-record change history derived from git and shown publicly as a trust feature. | **Decided — adopt** |
| 8 | Web-presence bias | Acknowledged in the methodology, with the counterpoint: publishing a curriculum requires no web design budget — a plain PDF suffices. Transparency is a legitimate demand on the school, not a website-quality contest; `not_published` therefore remains a fair finding. The inquiry workflow is the correction channel for schools that publish little but respond honestly. | **Decided — position taken** |
| 9 | Hosted trainings — provider vs. program boundary | A studio routinely lets an independent teacher run a training at its venue, where that teacher holds their *own* YA registration (specimen: Yoga Moves hosts both Marilou van Hoek's Ashtanga TT / "Pranayama" and Max Strom's Inner Axis). The record boundary follows the **canonical marketing + booking home**, not the venue: **own provider** when the training's primary page and enrolment live on its own brand site (Inner Axis → `maxstrom.com`); **program under the host** when the host is the primary marketer/booking channel (Ashtanga TT → `yogamoves.nl`), *even if* the certifying teacher has a separate site for *other* trainings (pranayama.nl markets her separate 100h mentorship, not the Ashtanga TT). In both cases the separate YA registration is recorded as a `holder` nuance and the venue relationship is cross-referenced between records. Rationale: §4.2 — `name` is the brand readers actually search for and book through. | **Decided (2026-06)** |
| 10 | YA registration monitor — separate longitudinal dataset | Build a standalone pipeline that periodically (monthly) snapshots YA RYS registrations for NL and records how they change, to make the lapse-and-renew pattern (a school/teacher letting a registration drop and re-registering only when needed) observable and citable — invisible on YA's live profile. Kept as a **separate, machine-owned dataset** (`ya-monitor/`, JSON, keyed by the stable YA school id), NOT in the curated provider schema; rule of thumb adopted: *generated = JSON, hand-curated = YAML*. Source is YA's public **guest Apex JSON API** (LWR `…/webruntime/api/apex/execute?asGuest=true`) + badge-PNG cross-check for designations — public, unauthenticated, no login/bot-bypass. Three layers: immutable `raw/` captures, overwritten normalized `snapshots/`, append-only `history/` event log. A delisting is recorded, never erased, and is an **earned finding**: only on a positive absence signal (hard 404 / N consecutive confirmed-absent runs), never on a single failed capture (`capture_failed` ≠ `delisted`) — the monitoring analogue of `not_published` vs `unknown`. History intervals are written at capture time and rendered at build time (static SSG has no git at runtime); git is the audit trail, not the query layer. Full design in `ya-monitor-spec.md`. **v0.2 (2026-06)** adds review capture + a teacher→training-origin lookup, derived as a confidence-tiered edge list — reviews leak the graduation school YA hides on teacher profiles (full reviewer name + linked member id + cohort dates), so capturing every school's reviews yields a reverse person→school index. Internal research aid only (human confirms before anything enters a curated record); review text stored as internal-only personal data; **gated on a ToS review before any capture, including the spike**. | **Decided — spec drafted (v0.2), unbuilt (2026-06)** |
| 11 | Host/venue without its own RYS — provider = the marketing+booking brand | Extends #9 to the case where the host studio holds **no YA RYS of its own** and every full TT it runs is certified by a rotating external RYS partner (specimen: **Bluebirds**, Amsterdam — does the marketing, booking, invoicing *and* CRKBO registration, but has no own RYS; its 2025 200h was certified by **MarleneYoga**'s RYS and its 2026 200h by **Embodied Bodhi Academy**'s RYS, a shell registered Dec 2025 for that collaboration). Resolution is the **same as #9**: the provider record is the **canonical marketing+booking brand** (Bluebirds), the trainings are **programs** under it (one per cohort/partner, a time series on the venue), and each certifying RYS is recorded as a `holder` nuance on its program + a `registrations` entry. A host lacking its own RYS does **not** demote it from being the provider — booking/marketing/CRKBO home is what counts. Consequence: the certifying RYS-holders get **no standalone provider record** (exactly as "Pranayama" never did under Yoga Moves) unless they independently market+book their own training elsewhere; record that the host has no own RYS as a `legal` note. The two host sub-cases now both resolve cleanly: (a) host *with* own RYS = Yoga Moves; (b) host *without* own RYS = Bluebirds. Rationale: §4.2 + #9 — `name` is the brand readers search and book through; the RYS paper is a back-end accreditation detail. | **Decided (2026-06)** |
| 12 | Price comparability — VAT normalization | Listings/filters compare on a **derived consumer-gross price (incl. btw)**, not the raw published `amount_eur`. Schools quote on three different VAT bases — `exempt_crkbo` (Tribes €3150, no btw), `incl` (Yoga Den €3597), `excl` (Wahé €2495) — so sorting on the raw figure is apples-to-oranges and, worse, rewards the (ACM-non-compliant) excl-btw display by making it look cheapest. Normalization rule: `incl` and `exempt_crkbo` → use as published (already the final price); `excl` → ×1.21 (NL yoga TT is standard-rated **21%**, not the 9% low rate); `vat: unknown` → **not comparable** — excluded from the price sort/filter and flagged "btw onbekend" (grossing up an unknown VAT status would be a guess, same `unknown` ≠ finding discipline as elsewhere). The gross is **derived in `dataset.ts`, never stored** (spec §6, like `pricePerContactHour`). Cards show the raw published price **plus** a VAT-basis label ("€2495 excl. btw ≈ €3019 incl.", "€3150 — geen btw (CRKBO)", "€3597 incl. btw") so the normalization is visible, not silent — which also surfaces the excl-btw oddity to the reader. Connects to the ACM price-display rule (consumer prices must be shown incl. btw; specimen + `acm-consumentenprijs` source on the Wahé record). Implement when the comparison UI is built. | **Decided — design fixed, unbuilt (2026-06)** |
| 13 | Yogic/dharma names — `Person.legal_name` | Yoga teachers commonly operate under a chosen spiritual (dharma) name. Added a dedicated **`Person.legal_name`** field (spec §4.7 + Zod) rather than folding both into `name`: `name` = the **publicly used** name (the dharma name if that's what they market, e.g. "Durga Devi", "Guru Gian"); `legal_name` = the civil/registered name when it differs and is **publicly self-disclosed or register-evidenced** (e.g. Durga Devi → Esther, from her own @durgadevi_esther handle). Rationale: the registers (YA/CRKBO/KvK/BIG) key on the civil name, so separating them preserves the verification anchor and keeps the brand-facing name intact (§4.2 logic, applied to people). Privacy posture: record `legal_name` only when the teacher has themselves made it public — the directory does **not** de-anonymize (AVG, §3). Bonus: a dharma-only public identity with **no** `legal_name` and **no** register trace is itself a transparency/verification-friction signal — it co-occurs with the unregistered/overclaim cluster (Yogaschool Noord, Adhouna, The Yoga Home). Specimen: `the-yoga-home` (Durga Devi/Esther + Kesharini=dharma-only). | **Decided — schema changed (2026-06)** |
